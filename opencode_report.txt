Task summary
------------
Implemented the first local LeRobot-style skeleton writer. The writer consumes a validated staged export, creates v2.1-style metadata folders, writes JSONL placeholder frame records in the future LeRobot-like data path, and stages image files into a controlled output tree by symlink or copy. No parquet was written. No videos were encoded. No packages were installed. No Hugging Face, ROS, LeRobot API, or any4lerobot integration was added.

Implemented behavior
--------------------
1. The default profile is forcevla_13d.
2. The secondary doosan_full_25d profile is supported by the same writer and validator framework.
3. The writer validates staged exports first with validate_staged_export.
4. The writer requires the CLI/profile argument to match metadata_staged.json profile.
5. The writer creates:
   output/meta/info.json
   output/meta/tasks.jsonl
   output/meta/episodes.jsonl
   output/meta/episodes_stats.jsonl
   output/data/chunk-000/episode_000000.jsonl
   output/image_staging/observation.image/episode_000000/
   output/image_staging/observation.wrist_image/episode_000000/
6. Frame records use future-facing keys:
   observation.image
   observation.wrist_image
   observation.state
   action
   timestamp
   frame_index
   episode_index
   task_index
   index
   task
7. observation.image and observation.wrist_image are relative paths to staged files under the skeleton output.
8. image-mode symlink creates absolute-target symlinks and fails clearly if symlink creation fails.
9. image-mode copy uses shutil.copy2.
10. Existing staged image targets are replaced if they are files or symlinks, and rejected if they are directories.
11. Terminal padding fields are not included in skeleton frame records.
12. info.json records skeleton_only true, parquet_written false, videos_encoded false, image_mode, and terminal_padding_excluded true.

Known limitations
-----------------
1. The skeleton data file is JSONL, not parquet.
2. No MP4 videos are encoded.
3. image shape is recorded as null because current dummy PPM images are placeholders and real camera dimensions are not known here.
4. The skeleton writer currently handles a single staged episode per output.
5. episodes_stats.jsonl is intentionally a stub with an empty stats object.
6. Symlink mode is intended for local/dummy development and is not portable if source images move.
7. Copy mode is more portable but still not a final training-ready LeRobot dataset.
8. No LeRobot package API is imported or used.
9. No dependency availability checks for pyarrow, pandas, video encoders, or lerobot were performed.
10. No Hugging Face upload path exists.
11. No any4lerobot integration exists.
12. No ROS recorder exists.

Exact next recommended task

Implement a real local LeRobot writer design decision step that checks available dependencies and confirms the exact v2.1 parquet/video schema before writing parquet or encoding videos. The next code should still avoid Hugging Face upload, ROS recording, package installation without approval, and any4lerobot integration until the local schema path is confirmed.

Final summary
-------------
The repository now has a complete offline path through local LeRobot-style skeleton export:

raw dummy episode -> validate raw -> convert to processed JSONL -> validate processed -> inspect processed -> plan dry-run export -> validate export plan -> stage export JSONL -> validate staged export -> write LeRobot skeleton -> validate LeRobot skeleton

The primary target remains forcevla_13d. doosan_full_25d remains supported as the secondary full-proprioception profile. The primary learning action label remains measured TCP delta from robot state, independent of demonstration method.
