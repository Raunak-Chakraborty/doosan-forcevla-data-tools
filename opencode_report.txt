Readiness logic
---------------
1. skeleton_valid is based on validate_lerobot_skeleton.
2. schema_valid checks profile, state/action dimensions, prompt/task features, prompt == task, terminal padding field absence, frame counts, and metadata flags.
3. image_staging_complete requires both observation.image and observation.wrist_image paths to exist under the skeleton root for every frame.
4. parquet_ready is true only when pyarrow is available.
5. video_ready is true only when ffmpeg is available and at least one of imageio, cv2, or PIL is available.
6. lerobot_api_available is true only when lerobot is importable.
7. real_export_ready is true only when skeleton/schema/images are valid and parquet_ready is true.
8. video_export_ready is true only when skeleton/schema/images are valid and video_ready is true.

Exit code policy
----------------
1. The CLI exits 0 when preflight checks run successfully and the skeleton validates, even if optional dependencies are missing.
2. The CLI exits nonzero when skeleton validation fails or the skeleton cannot be read.
3. The helper returns a structured report for schema problems, including prompt/task mismatches, so tests and callers can inspect bad exports without crashing.
