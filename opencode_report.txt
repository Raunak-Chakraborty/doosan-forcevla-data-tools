Current pipeline after this task
--------------------------------
The repository now supports this standard-library-only local pipeline:

raw dummy episode -> validate raw -> convert to processed JSONL -> validate processed -> inspect processed

The processed output remains:

data/processed_dummy/episode_000000/metadata_processed.json
data/processed_dummy/episode_000000/frames.jsonl

This is still not LeRobot parquet export, not Hugging Face integration, and not ForceVLA training code.

Core design preserved
---------------------
The dataset remains independent of demonstration collection method. The primary learning action label is still measured TCP delta computed from robot state:

action[t] = TCP pose delta from t to t + 1

The inspector and export plan do not require commanded_twist, commanded_action, predicted_action, or selected_action. Those streams can remain optional future auxiliary streams.

Updated export-plan decisions
-----------------------------
The concrete mapping document is now:

docs/lerobot_export_plan.md

It defines the current internal processed format:
1. metadata_processed.json
2. frames.jsonl
3. model_state 25D
4. measured_action 7D
5. image references

Export profile A: forcevla_13d
Purpose:
First compatibility target for official ForceVLA / tshiamor-like loading.

Mapping:
1. observation.image = external_rgb
2. observation.wrist_image = tcp_rgb
3. optional missing third camera = zero-filled later by ForceVLA transform or export profile
4. observation.state = 13D: ee_pos(3) + ee_axis_angle(3) + gripper_pos(1) + wrench(6)
5. action = measured_action 7D
6. task = task_instruction
7. final terminal-padded frame excluded from training/export by default

Export profile B: doosan_full_25d
Purpose:
Future full-proprioception experiments.

Mapping:
1. observation.image = external_rgb
2. observation.wrist_image = tcp_rgb
3. observation.state = full 25D model_state
4. action = measured_action 7D
5. task = task_instruction
6. final terminal-padded frame excluded from training/export by default

Image handling decision:
1. Current processed JSONL stores image references only.
2. Future export should either copy images into export staging before video encoding, or encode videos directly from referenced raw images.
3. Implementation choice is intentionally deferred.

any4lerobot decision:
1. Use only as a reference for LeRobot metadata and version conventions.
2. Do not clone, vendor, import, or integrate it until this repository's export mapping is stable.

Next implementation after this planning step:
Implement a dry-run export manifest generator, not parquet. It should report planned LeRobot keys, dimensions, selected profile, frame inclusion/exclusion, and image availability without writing videos or parquet files.

Known limitations
-----------------
1. No ROS recorder is implemented.
2. No LeRobot parquet export is implemented.
3. No Hugging Face integration is implemented.
4. No any4lerobot integration is implemented.
5. No package installation was performed.
6. The inspector is read-only and does not generate plots.
7. The inspector assumes the current v0 model_state layout for wrench indices 7 through 12.
8. The current processed format stores image path references and does not copy or encode images.
9. The converter still aligns streams by frame index only.
10. Advanced timestamp synchronization is not implemented.
11. The export plan is a mapping decision document only, not executable export code.
12. The final terminal-padded frame policy is planned as exclude-by-default for future export, but no export code implements that yet.

Exact next recommended task
---------------------------
Implement a dry-run export manifest generator for the planned export profiles.

Recommended scope:
1. Add a module such as src/doosan_forcevla_data/convert/plan_lerobot_export.py.
2. Accept a processed episode directory and an export profile name, starting with forcevla_13d.
3. Validate the processed episode first.
4. Exclude the final terminal-padded frame by default.
5. Print or write a small JSON manifest describing planned keys and dimensions.
6. For forcevla_13d, verify observation.state length 13 and action length 7.
7. For doosan_full_25d, verify observation.state length 25 and action length 7.
8. Verify image references exist.
9. Do not write parquet, encode videos, use Hugging Face, install dependencies, or integrate any4lerobot.

Bottom line
-----------
The repository now has a complete small v0 inspection loop for processed JSONL episodes and a concrete, non-executable export mapping plan. The next safe implementation step is a dry-run manifest generator before any LeRobot parquet export work.
