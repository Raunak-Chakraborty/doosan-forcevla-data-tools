Robustness issue 1: processed action nonzero checks
---------------------------------------------------
Previous behavior:
validate_processed_episode failed if no non-final action had nonzero translation or if no non-final action had nonzero rotation.

Problem:
That made a generic structural validator reject valid episodes that are stationary, near-stationary, constrained to pure translation, or constrained to pure rotation. A validator should check schema and numeric validity, not assume task motion content.

Change:
The hard errors were removed:
1. expected at least one non-final action with nonzero translation
2. expected at least one non-final action with nonzero rotation

Preserved behavior:
1. Action statistics remain in inspect_processed_episode.py.
2. Existing dummy tests still assert the dummy episode has nonzero translation and rotation.

Regression test:
tests/test_raw_to_processed.py now modifies processed non-final actions to have zero rotation and confirms validate_processed_episode still passes.

Robustness issue 2: export plan terminal padding count
------------------------------------------------------
Previous behavior:
validate_export_plan required excluded_terminal_padding_frame_count == 1.

Problem:
That was true for the current dummy convention but too specific for a generic export-plan validator. Future episodes or profiles might exclude zero, one, or more frames.

Change:
The validator now checks:
1. excluded_terminal_padding_frame_count is an integer.
2. excluded_terminal_padding_frame_count >= 0.
3. excluded_terminal_padding_frame_count == input_frame_count - exported_frame_count.
4. If terminal_padding_excluded is true, input_frame_count >= exported_frame_count.

Preserved behavior:
Dummy export-plan tests still assert the dummy value is 1.

Regression test:
tests/test_plan_lerobot_export.py now validates a synthetic manifest with input_frame_count 20, exported_frame_count 18, and excluded_terminal_padding_frame_count 2.

Robustness issue 3: staged export image path resolution
-------------------------------------------------------
Previous behavior:
stage_lerobot_export assumed image paths were directly usable as filesystem paths, which worked for absolute paths but not for relative paths.

Problem:
Processed JSONL may reasonably store image paths relative to the processed episode or relative to the source raw episode.

Change:
stage_lerobot_export now resolves image paths in this order:
1. If the path is absolute, require it to exist and use it.
2. Else try relative to the processed episode directory.
3. Else try relative to metadata_processed.json source_raw_episode.
4. If none exists, raise a clear ValueError.

Staged frames now write resolved absolute image paths for observation.image and observation.wrist_image.

Regression test:
tests/test_stage_lerobot_export.py now rewrites processed frames.jsonl image paths to source-raw-relative paths such as images/external_rgb/000000.ppm, then confirms staging resolves them to existing absolute paths.

Robustness issue 4: optional gripper_pos validation
---------------------------------------------------
Previous behavior:
tcp_pose.csv gripper_pos was optional, but if present it was not checked by validate_raw_episode.

Problem:
Non-finite values such as NaN or inf could pass raw validation if they appeared in an optional gripper_pos column.

Change:
validate_raw_episode now checks gripper_pos as a finite numeric field when the column exists. It still does not require gripper_pos to exist.

Regression test:
tests/test_dummy_raw_episode.py now writes NaN into gripper_pos and confirms raw validation fails with an error mentioning gripper_pos.

Robustness issue 5: dummy joint velocity consistency
----------------------------------------------------
Previous behavior:
Dummy joint velocity used total_joint_delta / (frame_count / fps), which is nominal frame coverage.

Problem:
For a 20-frame sequence starting at t=0, the timestamp span is (frame_count - 1) / fps, not frame_count / fps.

Change:
Dummy joint velocity now uses:

timestamp_span = (frame_count - 1) / fps
joint_velocity = total_joint_delta / timestamp_span

Regression test:
tests/test_dummy_raw_episode.py now asserts the default dummy joint_vel_0 is approximately 0.02 / (19 / 30).
