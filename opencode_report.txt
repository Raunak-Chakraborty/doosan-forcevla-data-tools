Current working pipeline
------------------------
The repository now supports this local standard-library-only pipeline:

raw dummy episode -> validate raw -> convert to processed JSONL -> validate processed -> inspect processed -> plan dry-run export -> validate export plan

The primary action label remains measured TCP delta from robot state:

action[t] = TCP pose delta from t to t + 1

No commanded action stream is required. This preserves compatibility with hand-guiding, teleoperation, SpaceMouse, VR controller, and future policy shadow-mode logs.

Inspector timing fix
--------------------
Previous behavior:

duration_seconds = frame_count / fps

For a 20-frame episode at 30 FPS, this printed 0.666666667 seconds. That value is nominal frame coverage, not timestamp duration. The actual timestamp span for frames starting at 0 is:

last_timestamp - first_timestamp = 19 / 30 = 0.633333333 seconds

New inspector summary fields:
1. fps
2. nominal_dt_seconds = 1 / fps if fps > 0 else 0
3. timestamp_start_seconds = first frame timestamp
4. timestamp_end_seconds = last frame timestamp
5. timestamp_span_seconds = timestamp_end_seconds - timestamp_start_seconds
6. nominal_frame_coverage_seconds = frame_count / fps if fps > 0 else 0

The printed output now shows all five timing fields and no longer prints frame_count / fps as duration_seconds.

Dry-run export planner details
------------------------------
Module:

src/doosan_forcevla_data/convert/plan_lerobot_export.py

CLI:

PYTHONPATH=src python3 -m doosan_forcevla_data.convert.plan_lerobot_export --processed data/processed_dummy/episode_000000 --profile forcevla_13d --output data/processed_dummy/episode_000000/export_plan_forcevla_13d.json

Supported profiles:
1. forcevla_13d
2. doosan_full_25d

Behavior:
1. Validates the processed episode first using validate_processed_episode.
2. Reads metadata_processed.json.
3. Reads frames.jsonl.
4. Excludes frames where action_is_terminal_padding is true.
5. Keeps terminal_padding_excluded = true in the manifest.
6. Counts excluded terminal padding frames.
7. Checks image availability for exported frames.
8. Writes a compact JSON manifest.
9. Prints a human-readable summary.
10. Does not write parquet.
11. Does not copy images.
12. Does not encode videos.
13. Does not import LeRobot.
14. Does not use Hugging Face.

forcevla_13d profile mapping
----------------------------
Purpose:
First compatibility target for official ForceVLA / tshiamor-like loading.

Mapping:
1. observation.image = external_rgb_path
2. observation.wrist_image = tcp_rgb_path
3. observation.state = first 13 values of model_state
4. observation.state layout = ee_pos(3) + ee_axis_angle(3) + gripper_pos(1) + wrench(6)
5. action = measured_action
6. task = task_instruction
7. expected observation_state_dim = 13
8. expected action_dim = 7
9. terminal padded final frame excluded by default

doosan_full_25d profile mapping
-------------------------------
Purpose:
Future full-proprioception experiments.

Mapping:
1. observation.image = external_rgb_path
2. observation.wrist_image = tcp_rgb_path
3. observation.state = full 25D model_state
4. action = measured_action
5. task = task_instruction
6. expected observation_state_dim = 25
7. expected action_dim = 7
8. terminal padded final frame excluded by default

Export plan validator details
-----------------------------
Module:

src/doosan_forcevla_data/validate/validate_export_plan.py

CLI:

PYTHONPATH=src python3 -m doosan_forcevla_data.validate.validate_export_plan data/processed_dummy/episode_000000/export_plan_forcevla_13d.json

Validation checks:
1. Manifest file exists.
2. Manifest is valid JSON.
3. Manifest root is a JSON object.
4. profile is forcevla_13d or doosan_full_25d.
5. exported_frame_count > 0.
6. input_frame_count > exported_frame_count when terminal padding is excluded.
7. excluded_terminal_padding_frame_count == 1 for the current v0/dummy convention.
8. required lerobot_like_keys exist.
9. observation_state_dim is 13 for forcevla_13d.
10. observation_state_dim is 25 for doosan_full_25d.
11. action_dim is 7.
12. image existing counts match exported_frame_count.
13. first_exported_record_preview observation_state_length matches expected state dim.
14. first_exported_record_preview action_length matches expected action dim.
